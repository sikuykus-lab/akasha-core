1. 术语定义
-----------

1.1 “您”“用户”：使用本产品的个人或组织。  
1.2 “商业使用”：以获取收入（营业收入/利润）为目的，对本产品或其衍生作品
     的任何使用行为（直接或间接），包括但不限于：
     - 作为付费服务、SaaS 平台、产品或功能的一部分；
     - 在公司内部服务中使用，并参与公司创收活动；
     - 在向第三方提供付费服务时使用本产品。
1.3 “非商业使用”：与创收活动无关的个人、教学或研究用途。  
1.4 “衍生作品”：在其中将本产品作为库、组件或重要组成部分使用的任何软件、
     修改或集成方案。

2. 非商业使用（个人和教学）
--------------------------

2.1 许可方免费授予您一项非独占权利，用于非商业目的：
     - 下载并阅读本产品的源代码；
     - 在本地运行本产品，用于个人或教学目的；
     - 为个人或教学目的修改您持有的本产品副本；
     - 通过 pull request 向原始代码仓库提交修改建议。

2.2 在非商业使用下，您不得：
     - 将本产品或其衍生作品作为托管服务（SaaS、API 等）提供给他人使用，
       即便是免费，只要该行为属于公司商业活动的一部分；
     - 出售、出租、转租、再许可本产品或其任何部分。

3. 无单独协议情况下的商业使用
------------------------------

3.1 如未与许可方签署单独的书面商业协议，您仅在 **同时满足以下全部条件**
     的前提下，方可在商业场景中使用本产品：

     a) 源代码公开  
        - 所有使用本产品的衍生作品（包括后端、前端、辅助服务和脚本）的
          完整源代码必须发布在公共代码仓库（GitHub 或类似平台），其许可
          协议不得弱于本许可（默认采用“传染性”/copyleft 思路）；  
        - 公共文档及 `README` 中必须明确标注使用了 AKASHA Core (`akasha-core`)，
          并附上指向许可方代码仓库的链接。

     b) 支付分成（版税）  
        - 用户必须向许可方支付 **总收入（gross revenue）的 5%** 作为版税，
          该收入包括所有使用本产品或其任何衍生作品的产品和服务所产生的
          全部营业收入；  
        - 版税结算及支付周期不得少于每自然季度一次。

3.2 如用户不希望公开衍生作品的全部源代码，或不愿遵守第 3.1 条的条件，则
     必须与许可方签署单独的商业协议，在该协议中另行约定授权条件及费用。

4. 报告与审计
-------------

4.1 在第 3 条所述商业使用情形下，用户须：
     - 对使用本产品的产品/服务单独核算收入；  
     - 在许可方提出书面请求（每年不超过一次）时，提供经汇总处理的收入报告，
       无需披露单个客户的商业机密。

4.2 许可方有权提前 30 个自然日书面通知后，委托独立第三方审计用户的版税
     计算情况：  
     - 如审计结果显示少付金额不超过 5%，审计费用由许可方承担；  
     - 如少付金额超过 5%，则所有审计费用由用户承担。

5. 违规、违约责任与许可终止
----------------------------

5.1 任何未遵守第 3 和第 4 条规定的商业使用，均视为对本许可的违反。

5.2 在用户违反本许可时，许可方有权要求：
     - 立即停止使用本产品及所有基于其构建的衍生作品；  
     - 补缴自开始商业使用以来全部应付而未付的版税；  
     - 追加支付最高不超过 **未付版税总额 100%** 的违约金；  
     - 赔偿许可方因该等违反行为所产生的全部可证明损失。

5.3 若在许可方发出书面通知（包括电子邮件或通过托管本仓库的平台发送）后
     30 个自然日内，用户仍未纠正违规行为，则本许可在该用户范围内自动终止。

6. 无担保条款
-------------

本产品按“现状”（“AS IS”）提供，许可方不对本产品作出任何明示或默示的保
证，包括但不限于对适销性、特定用途适用性以及不侵犯第三方权利的保证。在任
何情况下，许可方均不对因使用或无法使用本产品所引起或与之相关的任何索赔、
损失、损害或其他责任承担法律责任，无论该责任基于合同、侵权或其他法律理论。

7. 适用法律
-----------

本许可及由此产生或与之相关的任何争议，适用许可方在其公开信息或单独商业合
同中指定的国家/地区的实体法。如未作此指定，双方应首先通过协商解决争议；
如协商不成，则争议提交由许可方选择的有管辖权的法院审理。
